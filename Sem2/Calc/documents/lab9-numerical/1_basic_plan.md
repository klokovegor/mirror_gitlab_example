# Реализация `1_basic_tz.md`

## 1) Базовая структура решения

Для первичного ТЗ ядро вынесено в два публичных решателя в
`Sem2/Calc/src/numerical_methods.h`:

- `solve_bisection(...)`
- `solve_newton(...)`

Обе функции реализованы в `Sem2/Calc/src/numerical_methods.cpp` как самостоятельные
численные схемы, без внешних зависимостей на другие модули.

## 2) Что сделано в методе биссекции

В реализации `solve_bisection(...)` добавлены базовые шаги классического алгоритма:

- валидация интервала (`left <= right`);
- проверка существования корня на отрезке (смена знака на границах);
- итерационное деление интервала пополам;
- выбор нового подотрезка по знаку невязки;
- остановка при достижении условий завершения.

Также учтены крайние случаи:

- корень совпадает с левой/правой границей;
- функция возвращает некорректные значения (обрабатывается через безопасный вызов).

## 3) Что сделано в методе Ньютона

В `solve_newton(...)` реализована стандартная итерация:

- вычисление `f(x)` и `f'(x)`;
- переход к следующему приближению
  `x_next = x - f(x) / f'(x)`;
- контроль остановки по заданным критериям;
- защита от вырожденной производной (`|f'(x)|` слишком мало).

Для устойчивости добавлены проверки корректности входных параметров и промежуточных
вычислений.

## 4) Что уже заложено в интерфейс (как развитие базового ТЗ)

Хотя начальный пункт требовал просто найти корень, в текущей кодовой версии сразу заложена
расширяемость:

- структура настроек `method_settings_t`;
- структура результата `method_result_t`.

Это позволило в дальнейшем добавить критерии останова, диагностику и обработку ошибок без
переписывания базовых алгоритмов.

## 5) Тестовое покрытие по базовому ТЗ

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлены штатные тесты на поиск корней для
обоих методов на типовых функциях:

- линейное уравнение;
- квадратное уравнение;
- гиперболическая функция;
- гармоническая функция.

Тесты проверяют численную близость найденного корня к ожидаемому значению.
