# Реализация `7_validation_tz.md`

## 1) Что сделано концептуально

В проект добавлена численная верификация собственных реализаций методов поиска корня
через сравнение с эталонной библиотекой `fixed_solvers`.

Проверка оформлена как воспроизводимые автотесты:

- сравнение проводится для двух методов: `bisection` и `newton`;
- сравнение проводится на наборе уравнений из предыдущих задач;
- итоговый критерий: корни "нашего" и эталонного решателей близки с заданным допуском.

## 2) Что изменено в тестах (`test_numerical_methods.cpp`)

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлен блок валидации:

- `struct validation_case_t`
  - единое описание тестового кейса: имя, `residual`, `derivative`, границы и `x0`.
- `double solve_fixed_bisection_reference(...)`
  - обертка вызова `fixed_bisectional<1>::solve(...)` из `fixed_solvers`.
- `double solve_fixed_newton_reference(...)`
  - обертка вызова `fixed_newton_raphson<1>::solve(...)` из `fixed_solvers`.

Добавлены целевые тесты:

- `NumericalMethodsValidation.BisectionMatchesFixedSolvers`
  - сравнение `solve_bisection(...)` с эталонной биссекцией.
- `NumericalMethodsValidation.NewtonMatchesFixedSolvers`
  - сравнение `solve_newton(...)` с эталонным Ньютоном.

## 3) Покрытые уравнения и допуски

Оба теста запускаются на одинаковом наборе кейсов:

- линейное уравнение;
- квадратное уравнение;
- гиперболическое уравнение;
- гармоническое уравнение (`sin`/`cos`).

Использованные допуски:

- для биссекции: `tolerance = 1e-6`;
- для Ньютона: `tolerance = 1e-5`.

При расхождении тест выводит расширенную диагностику:

- имя кейса;
- метод;
- `our_root`;
- `reference_root`;
- `abs_error`.

## 4) Что изменено в CMake (`Sem2/Calc/CMakeLists.txt`)

Добавено подключение установленного пакета `fixed_solvers`:

- `find_package(fixed_solvers REQUIRED CONFIG ...)`;
- линковка `fixed_solvers::fixed_solvers` к цели `Tests`.

Это позволяет использовать API эталонных методов напрямую из тестов.

## 5) Что важно про минимальность изменений

- вычислительная логика `numerical_methods.cpp/.h` не изменялась;
- валидация добавлена только в тестовый слой;
- интеграция с `fixed_solvers` выполнена через CMake и тестовую цель.

## 6) Итог

- Требование ТЗ по численному сравнению с `fixed_solvers` реализовано.
- Сравнение покрывает ключевые уравнения из предыдущих этапов лабораторной.
- Использование эталонных методов теперь автоматизировано и проверяется тестами.
