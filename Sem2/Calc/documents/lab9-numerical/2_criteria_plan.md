# Реализация `2_criteria_tz.md`

## 1) Изменения в публичном интерфейсе

В `Sem2/Calc/src/numerical_methods.h` для управления остановкой добавлены:

- `enum class stop_criterion_t`:
  - `by_argument`,
  - `by_residual`,
  - `by_iterations`.
- `struct method_settings_t`:
  - `epsilon` — общий числовой порог;
  - `use_epsilon_x` — включить критерий по аргументу;
  - `use_epsilon_r` — включить критерий по невязке;
  - `iteration_count` — лимит итераций.
- `struct method_result_t`:
  - `root`,
  - `residual`,
  - `iterations`,
  - `stop_criterion`.

Итог: пользователь получает не только корень, но и явную причину завершения метода.

## 2) Как критерии реализованы в алгоритмах

В `Sem2/Calc/src/numerical_methods.cpp` обе функции (`solve_bisection`, `solve_newton`)
проверяют критерии в процессе итераций:

- критерий по аргументу:
  - для Ньютона — по `|x_next - x|`,
  - для биссекции — по текущей ширине интервала;
- критерий по невязке:
  - по `|r(x)|`;
- критерий по числу итераций:
  - срабатывает при достижении лимита.

Результат формируется через `method_result_t` с корректным `stop_criterion`.

## 3) Режимы включения критериев

Поддержаны все режимы, требуемые ТЗ:

- только по аргументу (`use_epsilon_x=true`, `use_epsilon_r=false`);
- только по невязке (`use_epsilon_x=false`, `use_epsilon_r=true`);
- оба сразу (`true/true`) — остановка по первому достигнутому условию.

Добавлена защита от некорректного режима:

- если оба критерия отключены (`false/false`) — выбрасывается `std::runtime_error`.

## 4) Валидация параметров

Для предотвращения неустойчивых сценариев добавлены проверки:

- `epsilon` должен быть конечным и положительным;
- `iteration_count > 0`.

Это сделано одинаково для обоих методов.

## 5) Тесты критериев останова

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлены отдельные тесты
`StopsByArgumentCriterion`, `StopsByResidualCriterion`, `StopsByIterationsCriterion`
для биссекции и Ньютона.

Под каждую проверку подобраны исходные данные так, чтобы срабатывал именно целевой критерий.
Это исключает ложные прохождения из-за другого условия остановки.
