# Реализация `6_polymorphism_tz.md`

## 1) Что изменилось концептуально

Код переработан от "двух независимых функций `r` и `dr`" к полиморфной модели уравнения:

- связь невязки и производной теперь выражается одним типом `root_equation_t`;
- расчетчик принимает абстракцию уравнения и работает как "черный ящик" для пользователя;
- добавлена единая точка запуска `solve_root(...)` с выбором метода.

Это делает интерфейс очевиднее и лучше соответствует LSP.

## 2) Изменения в интерфейсе (`numerical_methods.h`)

В `Sem2/Calc/src/numerical_methods.h` добавлены:

- `enum class root_method_t`
  - `bisection`
  - `newton`
- `class root_equation_t`
  - `virtual double residual(double x) const = 0;`
  - `virtual double derivative(double x) const;`
  - виртуальный деструктор.
- `method_result_t solve_root(...)`
  - единая точка запуска для пользователя.

Текущие `solve_bisection(...)` и `solve_newton(...)` сохранены как базовые вычислительные блоки.

## 3) Реализация полиморфного слоя (`numerical_methods.cpp`)

В `Sem2/Calc/src/numerical_methods.cpp` сделано:

- Реализован `root_equation_t::derivative(double x) const`:
  - по умолчанию использует `make_numerical_derivative(...)`;
  - тем самым производная автоматически доступна даже если пользователь не переопределил её в наследнике.
- Реализован `solve_root(...)` как роутер:
  - при `root_method_t::bisection` вызывает `solve_bisection(...)` через `equation.residual`;
  - при `root_method_t::newton` вызывает `solve_newton(...)` через `equation.residual` и `equation.derivative`.
- Добавлена валидация входов в зависимости от метода:
  - для биссекции должны быть заданы `left/right`;
  - для Ньютона должен быть задан `x0`.

Важно: численная математика итераций не переписана, переиспользуется существующий код.

## 4) Почему это соответствует LSP и требованию "жесткой связи r/dr"

- Пользователь передает объект, реализующий контракт `root_equation_t`.
- Любая конкретная реализация уравнения подставляется взаимозаменяемо:
  - линейная,
  - гармоническая,
  - любая другая.
- Связь `r` и `dr` закреплена в одном интерфейсе уравнения, а не во внешней паре функций.

## 5) Тесты полиморфизма

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлены тесты:

- `NumericalMethodsPolymorphism.SolveRootBisectionForLinearEquation`
  - проверяет единый запуск для биссекции на линейном уравнении.
- `NumericalMethodsPolymorphism.SolveRootNewtonUsesDefaultNumericalDerivative`
  - проверяет единый запуск для Ньютона на гармоническом уравнении;
  - дополнительно подтверждает, что дефолтная численная производная работает корректно.

Для этого в тестах введены два класса-наследника:

- `linear_equation_t` (с явной аналитической производной),
- `harmonic_equation_t` (только `residual`, производная берется по умолчанию).

## 6) Итог по реализации

- Добавлен полиморфный интерфейс уравнения.
- Добавлена единая точка запуска вычислений.
- Старая логика критериев/диагностики/обработки ошибок сохранена и переиспользована.
- Добавлены целевые тесты на два разных вида уравнений для подтверждения работоспособности.
