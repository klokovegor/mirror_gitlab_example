# Реализация `3_exceptions_tz.md`

## 1) Принятая стратегия обработки ошибок

Реализован подход `fail-fast`:

- как только обнаружены нечисловые значения (`NaN/Inf`) или выброшено исключение из
  пользовательской функции, расчет немедленно прекращается;
- метод не продолжает итерации после ошибки;
- наружу всегда выходит `std::runtime_error` с объясняющим контекстом.

Это соответствует требованию ТЗ: не допускать «ложных» результатов при поврежденном ходе вычислений.

## 2) Центральная точка обработки

В `Sem2/Calc/src/numerical_methods.cpp` выделена функция:

- `evaluate_function(...)`

Она выполняет:

- безопасный вызов `std::function<double(double)>`;
- проверку `std::isfinite(value)` для результата;
- формирование контекста вида:
  - какой метод (`bisection/newton/derivative`);
  - какой этап (`left-boundary`, `midpoint`, `derivative-value` и т.д.);
  - номер итерации;
- перехват исключений:
  - `std::exception`,
  - неизвестные исключения.

## 3) Поведение при разных типах ошибок

### Нечисловой результат

- При `NaN/Inf` формируется `std::runtime_error` с контекстом этапа.

### Исключение из пользовательской функции

- Исключение перехватывается;
- пробрасывается `std::runtime_error` с добавленным контекстом вызова;
- исходное сообщение (`e.what()`) включается в текст ошибки.

## 4) Где это используется

`evaluate_function(...)` используется как единый механизм в:

- `solve_bisection(...)`:
  - границы интервала;
  - midpoint каждой итерации.
- `solve_newton(...)`:
  - `f(x)`;
  - `f'(x)`;
  - финальные/дополнительные проверки.
- `make_numerical_derivative(...)`:
  - `r(x + h)`, `r(x - h)`.

Таким образом, поведение при ошибках единообразно по всем сценариям.

## 5) Тесты отказоустойчивости

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлены негативные тесты:

- для биссекции:
  - `NaN` из `r(x)`,
  - исключение из `r(x)`;
- для Ньютона:
  - `NaN` из `r(x)`,
  - `NaN` из `dr(x)`,
  - исключение из `r(x)`,
  - исключение из `dr(x)`.

Во всех случаях проверяется выброс `std::runtime_error`.
