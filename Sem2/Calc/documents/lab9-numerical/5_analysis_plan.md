# Реализация `5_analysis_tz.md`

## 1) Что изменилось в публичном API

Для поддержки диагностики без отдельного класса-логгера были расширены уже существующие структуры в
`Sem2/Calc/src/numerical_methods.h`.

- Добавлена структура точки трассы:
  - `iteration_trace_point_t`
  - Поля:
    - `iteration` — номер итерации;
    - `x` — значение аргумента на этой итерации;
    - `residual` — значение невязки `r(x)`;
    - `delta_x` — шаг по аргументу (для анализа скорости сходимости).
- Добавлена структура общей диагностики:
  - `diagnostics_t`
  - Поля:
    - `history` — массив точек итераций (`std::vector<iteration_trace_point_t>`);
    - `is_monotonic_convergence` — итоговый признак монотонной сходимости.
- В `method_settings_t` добавлены настройки:
  - `collect_diagnostics` — включить/выключить сбор истории;
  - `diagnostics_reserve_hint` — подсказка для `reserve(...)` вектора истории.
- В `method_result_t` добавлено поле:
  - `diagnostics` — полностью собранная диагностическая информация.

Это позволяет пользователю получить историю итераций прямо из результата метода и сразу строить графики.

## 2) Как встроен сбор диагностики в вычисления

В `Sem2/Calc/src/numerical_methods.cpp` добавлены служебные функции:

- `init_diagnostics(...)`:
  - создает объект `diagnostics_t`;
  - при включенной диагностике заранее делает `reserve(...)`, если задан `diagnostics_reserve_hint`;
  - сразу выставляет базовое значение флага монотонности.
- `append_diagnostics_point(...)`:
  - добавляет точку в `history` только если `collect_diagnostics == true`;
  - одновременно обновляет признак монотонности.

### Для биссекции

- В историю пишутся:
  - стартовые граничные точки (левый/правый конец);
  - midpoint на каждой итерации.
- В `delta_x` для биссекции используется текущая ширина интервала.

### Для Ньютона

- В историю пишется каждая итерация с текущим `x` и `r(x)`.
- `delta_x`:
  - на первой итерации `0.0`;
  - далее `|x_k - x_{k-1}|`.
- При остановке по аргументу дополнительно фиксируется финальная точка `x_next`.

## 3) Как считается монотонная сходимость

Признак монотонности рассчитывается онлайн в `append_diagnostics_point(...)`:

- Сравниваются соседние значения `|residual|`.
- Если текущее `|residual|` стало больше предыдущего больше чем на `monotonic_tolerance`,
  флаг `is_monotonic_convergence` становится `false`.
- Используется допуск `monotonic_tolerance = 1e-12`, чтобы не ломать монотонность из-за
  численного шума `double`.

Таким образом, критерий монотонности работает устойчиво и не зависит от случайных микроколебаний.

## 4) Баланс удобства и производительности

Реализован следующий баланс:

- Типовой сценарий (бизнес-логика): диагностика выключена.
  - Нет накопления истории;
  - Нет лишних `push_back`;
  - Нет лишних аллокаций.
- Режим анализа: диагностика включается явно.
  - Пользователь получает подробный трейс для визуализации;
  - Можно анализировать скорость/характер сходимости.

Ограничение ТЗ соблюдено: используется только стандартный `std::vector`, прямой работы с памятью нет.

## 5) Что проверено тестами

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлен отдельный набор тестов
`NumericalMethodsDiagnostics`:

- `DisabledDiagnosticsKeepsHistoryEmpty`
  - Проверяет, что при выключенной диагностике `history` пустой.
- `EnabledDiagnosticsCollectsHistoryPoints`
  - Проверяет, что при включении диагностики история действительно собирается;
  - Дополнительно проверяет корректность базовых полей точки (`x/residual/delta_x` конечны).
- `MonotonicConvergenceTrueForLinearNewton`
  - Проверяет сценарий монотонной сходимости (`true`).
- `MonotonicConvergenceFalseForBadDerivative`
  - Проверяет сценарий немонотонной динамики (`false`).

После внесения изменений весь набор тестов проекта проходит.
