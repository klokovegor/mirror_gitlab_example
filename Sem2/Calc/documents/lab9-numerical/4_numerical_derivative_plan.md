# Реализация `4_numerical_derivative_tz.md`

## 1) Архитектурное решение без дублирования Ньютона

Выбран вариант, где алгоритм Ньютона остается единым:

- используется существующий `solve_newton(r, dr, x0, settings)`;
- источник производной (аналитическая/численная) выбирается снаружи.

Для этого добавлена отдельная пользовательская функция:

- `make_numerical_derivative(...)`.

Пользовательский сценарий:

- `auto dr = make_numerical_derivative(r, eps);`
- `auto result = solve_newton(r, dr, x0, settings);`

Это устраняет дублирование кода итерационного метода и соответствует уточнению ТЗ.

## 2) Что добавлено в API

В `Sem2/Calc/src/numerical_methods.h` добавлена фабрика:

- `std::function<double(double)> make_numerical_derivative(const std::function<double(double)>& r, double h = 1e-8);`

Параметр `h` трактуется как базовый `eps` для вычисления локального шага.

## 3) Формула численной производной

В `Sem2/Calc/src/numerical_methods.cpp` используется центральная разность:

- `r'(x) ~= (r(x + h_x) - r(x - h_x)) / (2 * h_x)`.

Локальный шаг выбирается адаптивно (по уточнению ТЗ):

- `h_x = eps * max(1, |x|)`.

То есть:

- при `|x| < 1` шаг равен `eps`;
- при `|x| >= 1` шаг масштабируется величиной аргумента.

## 4) Валидация и устойчивость

Реализованы проверки:

- `eps` должен быть конечным и положительным;
- результат производной должен быть конечным.

Для вычисления `r(x + h_x)` и `r(x - h_x)` используется уже существующая безопасная обвязка
`evaluate_function(...)`, поэтому:

- NaN/Inf обрабатываются единообразно;
- исключения из `r(...)` перехватываются и пробрасываются с контекстом.

## 5) Тесты

В `Sem2/Calc/test/test_numerical_methods.cpp` добавлены тесты:

- `NumericalDerivativeMatchesAnalyticalValue`
  - проверка точности численной производной в контрольной точке против аналитической;
- `SolvesQuadraticEquationWithUserNumericalDerivative`
  - проверка, что Ньютон корректно работает с производной от фабрики;
- `ThrowsOnNonPositiveStep`
  - проверка ошибки на невалидный шаг (`0` и отрицательный).

Это покрывает и корректность формулы, и рабочий пользовательский сценарий.
